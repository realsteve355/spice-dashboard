## 1. AToken.sol → Full ERC-721 (Option B)

### Motivation
Users want A-tokens to be visible as real NFTs in MetaMask and verifiable on Basescan.
The old AToken was a custom registry (no ERC standard) — invisible to wallets.

### What changed
- `AToken.sol` now inherits OZ v5 `ERC721("SPICE A-Token", "ATOKE")`
- Internal `_mint` renamed → `_createToken`; `_transfer` → `_moveToken` (avoid OZ name clash)
- `_createToken` calls `super._mint(holder, id)` after registry write
- `_moveToken` calls `super._transfer(from, to, id)` after registry write
- `_deactivate` calls `super._burn(id)` after removing from held index
- Public `transferFrom`, `safeTransferFrom`, `approve`, `setApprovalForAll` → revert ("use Colony")
- `tokenURI(id)` returns `data:application/json;base64,...` on-chain JSON with form-specific attributes
- Token IDs start at 1 (constructor: `nextId = 1`)

## 2. Assets.jsx — New Page

**Route:** `/colony/:slug/assets`

Two tabs:

**Assets tab:**
- Lists all UNILATERAL A-tokens held by connected wallet
- Shows: name (localStorage), declared value, current depreciated value, weight, AI flag
- Inline transfer form (enter recipient address + new price)
- Register form: name, value, weight, AI checkbox, depreciation rate — validated against threshold (>500 S or >50 kg or AI)

**Obligations tab:**
- Lists all OBLIGATION_ASSET (creditor) and OBLIGATION_LIABILITY (obligor) tokens
- Progress bar (epochs paid / total epochs)
- Monthly amount, remaining epochs, collateral (if secured)
- Create obligation form: creditor address, obligor address, monthly amount, total epochs; optional collateral selector from your unilateral assets

**Asset names** are stored in `localStorage['spice_asset_names']` keyed by `colonyAddr:tokenId` —
there is no name field in the AToken contract.

**Navigation:** Dashboard has an "ASSETS & OBLIGATIONS" card that navigates to this page.

## 4. Vesting Display Fix — Root Cause

Previous session issue: Carla's vesting shares showed "Fully vested" immediately after issuance.
Root cause: the deployed AToken bytecode pre-dated the `hasVesting ? 0 : stakeBps` fix —
`vestedBps` was always set equal to `totalStakeBps` regardless of vesting schedule.
Fix: full colony redeploy with corrected AToken.sol (this session's deploy).
